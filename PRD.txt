# Job Tracker — Product Requirements Document

## Overview

Build a full-stack **Job Application Tracker** web app. Users can record job applications, track hiring stages, log interviews, and analyze their job search progress.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js (App Router), TypeScript, TailwindCSS, TanStack Query |
| Backend | NestJS, TypeScript |
| Database | PostgreSQL |
| ORM | Prisma |

---

## Core Features

### 1. Authentication
- User registration, login, logout
- JWT stored in **httpOnly cookies**
- Passwords hashed with **bcrypt**
- All routes protected — users can only access their own data
- Input validation via `class-validator`

### 2. Job Applications (CRUD)
Each application record contains:

| Field | Type |
|---|---|
| company | string |
| jobTitle | string |
| description | string |
| location | string |
| jobLink | string |
| salaryRange | string |
| applicationDate | date |
| status | enum (see below) |
| notes | string |

**Status enum (stored in DB):**
```
WISHLIST | APPLIED | INTERVIEW | OFFER | REJECTED | ACCEPTED
```

### 3. Interview Tracking
- Each application can have **multiple** interviews
- Interview fields: `stage`, `date`, `notes`
- Example stages: Recruiter call, Technical, System Design, Final

### 4. Filtering & Search
Filter applications by: `status`, `company`, `dateApplied`
Search by: `company`, `jobTitle`

### 5. Dashboard
Display:
- Total applications
- Applications count per status
- Recent applications
- Upcoming interviews

## API Endpoints

### Auth
```
POST   /auth/register
POST   /auth/login
GET    /auth/me
```

### Applications
```
GET    /applications              (paginated: ?page=1&limit=10)
GET    /applications/:id
POST   /applications
PATCH  /applications/:id
DELETE /applications/:id
```

### Interviews
```
POST   /applications/:id/interviews
GET    /applications/:id/interviews
```

### Analytics
```
GET    /analytics/summary
```

### API Response Format
```json
// Success
{ "data": {}, "message": "success" }

// Error
{ "statusCode": 400, "message": "...", "error": "Bad Request" }
```

## Database Schema

### Models

**User**
- id, email, passwordHash, createdAt, updatedAt
- Relations: has many Applications

**Application**
- id, userId (FK), company, jobTitle, description, location, jobLink, salaryRange, applicationDate, status (enum), notes, createdAt, updatedAt
- Relations: belongs to User, has many Interviews, has many Tags

**Interview**
- id, applicationId (FK), stage, date, notes, createdAt

**Tag**
- id, name
- Relations: many-to-many with Application

### Indexes
- `Application.userId`
- `Application.status`
- `Application.createdAt`

## Non-Functional Requirements

| Requirement | Spec |
|---|---|
| Rate limiting | 100 requests/minute |
| Pagination | All list endpoints (page + limit) |
| Type safety | TypeScript strict mode throughout |
| Linting | ESLint |
| DB queries | Avoid over-fetching; select only needed fields |
